PRG Ming : Sprint 3

Created by Unbekannter Benutzer (jacqueline.dittrich@fh-kiel.de), last modified by Jacqueline Rähse on Sep 23, 2021

Sprintziel


Das Sprintziel besteht aus 2 Teilen: Theorie & Praxis. Dabei baut die Praxis auf die Theorie auf. Soll heißen: wenn ihr die Theorie gelernt und verinnerlicht habt, sollte die Anwendung in der Praxis kein Problem sein.


Lernziel


Jedes Teammitglied  hat die Lerninhalte zu Sprint 3 verstanden und kann Programmieraufgaben zu den Themen (wie z.B. die Übungsaufgaben) lösen.

Projektziel

Ihr wählt als Team in der Planung eures Sprints selbstständig Projektanforderung aus, die ihr in diesem Sprint schaffen möchtet. Richtwert sind ca. 4-5 Anforderungen. Diese Anforderungen sollen dann am Ende des Sprints erfüllt sein. Jedes Teammitglied sollte pro Sprint für eine Anforderung verantwortlich sein.


Programmierstil

Auch der Programmierstil wird bewertet. Achtet darauf, dass euer Code eine hohe Qualität hat. Achtet hierbei auf folgende Punkte:

  • Lesbarkeit & Nachvollziehbarkeit: Würde auch eine andere Person euren Code einfach lesen und verstehen können?
    • ausreichend Kommentare, sinnvolle Bezeichner, Konventionen einhalten
  • Struktur & Übersichtlichkeit: Finde ich mich bzw. auch eine andere Person sich gut im Code zurecht?
    • Code sinnvoll auf Tabs aufteilen
    • Wenig globale Variablen verwenden. Grundsatz: So wenig wie möglich, so viele wie nötig.
    • Mit möglichst wenig Codezeilen auskommen, nichts mehrfach schreiben (dann lieber als Funktion)
  • Stabilität
    • Programm stürtzt nicht ab und macht, was es soll
